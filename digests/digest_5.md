## Joseph

:::{epigraph}
- Why does taking b + b.T automatically produce a symmetric matrix? 

- Is symmetry the only requirement for a matrix to represent an undirected graph?
:::


- A matrix A is symmetric if A.T = A. If A = b + b.T, then A.T = (b + b.T).T = (b.T + b.T.T) = (b.T + b) = A, 
  so A is symmetric. 

- Every symmetric matrix consisting of ones and zeros is an adjacency matrix 
  of some undirected graph.

## Alyssa

:::{epigraph}
What does the seed of a function tell you? Is it somewhat the coordinates of the function?
:::

Seed can be used with a random number generator and it provides a starting point for the generator. 
If you specify a seed, then every time you execute the code, the random number generator will produce 
the same sequence of randomly looking numbers. If you don't specify a seed, then each time the numbers
produced by the random number generator will be different. Seeds are useful when in some computations 
we want numbers that look random, but at the same time we want results of computations 
to be reproducible  - e.i. the same each time someone run the code.
